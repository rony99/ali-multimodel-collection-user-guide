## 参与方式

详见 **[参与方式.md](./参与方式.md)**。摘要：

- **对象**：理工类程序员相关从业者（前后端、数据分析、测试、DevOps 等）  
- **门槛**：Claude Code 完成过 ≥2 个项目；有 Docker 与单元测试经验；已读操作步骤，疑问经 qa_skill 后仍有兴趣  
- **报名**：按文档模板编辑材料，微信发给 **栗子**，进入项目参与群  
- **开跑后**：开放 Gateway，手机号注册，领取 Opus / GLM / 千问 配置（我们提供 API，不可用自备模型）

## FAQ

**Q：可以用自己的模型 / 自己的 API 吗？**  
A：不可以。**我们会提供 API**（及连接配置），须用平台指定的三套模型采集，不能换成自备 API 或本地模型。正式开始后通过 Gateway 手机号注册领取配置。

**Q：怎么才算「通过」？**  
A：单次看 Docker 里跑 `tests/test.sh` 是否退出码 `0`。题目是否最终合格，还要看操作步骤里的汇总门槛，并由后续评定。

**Q：「一条数据」是某一个模型跑一次吗？**  
A：不是。一条 = **一道完整题**（甲方数据包 + 三模型多次 session），不是「某一个模型跑一次」。

**Q：预检通过了就算过关吗？**  
A：不算。预检只做简单检测；最终通过与否需要后续评定。

**Q：怎么报名参与？**  
A：满足 [参与方式.md](./参与方式.md) 基本要求后，按模板微信发给 **栗子** 进群。
