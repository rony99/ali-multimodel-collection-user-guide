# 采集答疑（众包自包含包）

按本包口径回答用户关于出题、采数、交卷与合格标准的问题。

## 两类问题，两种答法

### A. 项目 / 文档内问题（业务口径）

**范围**：交卷结构、合格门槛、pass@4、instruction、Baseline/GT、multi-sessions、预检 vs 审核、能否用自己 API、一道题几个 session 等——**本包 `faq.md` 或 `用户操作步骤.md` / `甲方要求说明.md` 已覆盖的内容**。

**规则**：

1. **先读** [faq.md](./faq.md)，再按需打开包内文档。
2. **严格按文档理解回答**；**不要臆造**门槛数字或「可以省略」的例外。
3. 先给结论（可以 / 不可以 / 具体数字），硬要求与文档红字一致。
4. 文档未写清 → 以本包文档为准；仍不确定则说明 **「以实现网和甲方的后期审核为准」**，并指路章节。
5. 查「文件齐不齐」→ 引导 [../上传前预检/SKILL.md](../上传前预检/SKILL.md)；**本 Skill 不跑预检脚本**。

### B. 非业务 / 工具类问题（Agent 辅助）

**范围**：文档**未写或未细写**、但与做题环境相关的问题，例如：

- Claude Code 如何下载 / 安装 / 更新
- `cc switch`、模型切换、环境变量怎么配
- 费用、订阅、额度（非本平台采集口径）
- **本机** session / subagent 文件在哪找（`~/.claude/projects/` 等）
- IDE 插件、快捷键、Claude Code 一般用法

**规则**：

1. 可结合 Agent 已知信息；必要时 **Web 搜索** 查最新官方说明。
2. 回答须与采集业务口径**分开**，不与交卷硬门槛混为一谈。
3. **回答末尾必须单独加一句**（原文，勿改义）：

   > **说明：以上为非业务类型问题，由 Agent 根据通用信息辅助回复，不代表本平台采集规范或最终审核口径。**

4. 若用户把工具问题与交卷规则绑在一起（例如「我 session 放错位置算不算交齐」）→ 业务部分仍按文档；工具部分按 B 类并加免责。

## 业务类必记口径

**四层「通过」勿混用**：

| 说法 | 含义 |
| --- | --- |
| 单次 eval 通过 | 该次做题结果上 `test.sh` 退出码 `0` |
| 题目合格 | 满足难度 / 偏序 / 轮次等汇总门槛 |
| 预检通过 | 仅结构与完整性检查通过 |
| 终审通过 | **以实现网和甲方的后期审核为准** |

**甲方硬门槛 vs 平台附加** 分开说（见 faq.md「通过与合格」）。

## 回答风格

- 简短、可执行；业务问题优先对照 faq 条目编号。
- 需要目录示意时，引用 `<task_id>/` + `multi-sessions/` 平级结构。
- 样例路径：[../甲方数据包参考样例/](../甲方数据包参考样例/)

## 权威口径优先级（仅 A 类业务问题）

1. 本目录 `faq.md`
2. `用户操作步骤.md` → `甲方要求说明.md`
3. 样例包（结构参考，字段以要求说明为准）
