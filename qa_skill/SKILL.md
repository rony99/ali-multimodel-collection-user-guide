# 采集答疑（众包自包含包）

按本包口径回答用户关于出题、采数、交卷与合格标准的问题。

## Agent 必须遵守

1. **先读** [faq.md](./faq.md)，答案以之为准。细节不够再打开：
   - [../用户操作步骤.md](../用户操作步骤.md)（主线操作）
   - [../甲方要求说明.md](../甲方要求说明.md)（门槛 / 字段补充）
   - **不要臆造**门槛数字或「可以省略」的例外。
2. **先给结论**（可以 / 不可以 / 具体数字），再给短解释；硬要求与文档红字一致。
3. **四层「通过」勿混用**：
   | 说法 | 含义 |
   | --- | --- |
   | 单次 eval 通过 | 该次做题结果上 `test.sh` 退出码 `0` |
   | 题目合格 | 满足难度 / 偏序 / 轮次等汇总门槛 |
   | 预检通过 | 仅结构与完整性检查通过 |
   | 终审通过 | **以实现网和甲方的后期审核为准** |
4. **甲方硬门槛 vs 平台附加** 分开说（见 faq.md「通过与合格」）。
5. 口径冲突或文档未写清：以本包文档为准；仍不确定则明确说  
   **「以实现网和甲方的后期审核为准」**，并指出应查阅的章节。
6. 用户要查「文件齐不齐 / 目录对不对」→ 引导执行 [../上传前预检/SKILL.md](../上传前预检/SKILL.md)；**本 Skill 不跑预检脚本**。

## 回答风格

- 简短、可执行；优先对照 faq 条目编号。
- 需要目录示意时，引用 README / 操作步骤中的 `<task_id>/` + `multi-sessions/` 平级结构。
- 样例路径：[../甲方数据包参考样例/](../甲方数据包参考样例/)

## 权威口径优先级

1. 本目录 `faq.md`
2. `用户操作步骤.md` → `甲方要求说明.md`
3. 样例包（结构参考，字段以要求说明为准）
